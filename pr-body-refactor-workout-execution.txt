### Task A — Responsibility Map (pre-refactor)

**useState / useRef**
- `drafts: ExerciseDraft[]` — per-set UI state (weight/reps/rpe/done) for all exercises
- `sessionNotes: string` — text in the notes TextInput
- `saving: boolean` — true while handleSubmit is awaiting Firestore
- `saveError: string | null` — user-visible error from handleSubmit
- `message: string | null` — transient success message (never visible; nav happens immediately)
- `planRef` — sync ref to `plan` for stale-closure–safe access in callbacks
- `draftsRef` — sync ref to `drafts` for stale-closure–safe access in callbacks
- `sessionInitRef` — prevents double-initialization when plan loads
- `notesDebounceRef` — debounce timer for notes → context sync
- `inputRefs` — 2D map of TextInput refs for cross-row focus management [exIdx][setIdx]

**useEffect**
1. Dep `[workoutPlanId, authUserId, authUserRole]` — resets `sessionInitRef.current = false` so init runs again when user/plan changes
2. Dep `[plan?.id]` — session init: restore existing session OR start fresh OR redirect to active session
3. Dep `[]` (unmount) — clears the notes debounce timer

**Async functions**
- `fetcher` (useCallback) — loads plan via `getWorkoutPlanById`, loads history via `getWorkoutHistory` (sequential); throws on auth failure, missing plan, or access violation
- `handleSubmit` — validates all set drafts, computes per-exercise volume / PR detection / 1RM data, calls `workoutService.logCompletedWorkout`, calls `activeWorkout.finishSession()`, alerts on PRs, navigates to workoutHistory

**JSX sections**
- Lines 341–347: loading spinner (full-screen)
- Lines 349–355: load error (full-screen)
- Lines 357–363: plan-not-loaded fallback
- Lines 365–731: main ScreenLayout containing:
  - Lines 379–431: header (back button, title, elapsed timer badge, plan name, student name)
  - Lines 432–463: session notes card (TextInput)
  - Lines 465–696: exercise rows loop (exercise header + set table header + set rows per exercise)
  - Lines 698–703: save error / success message
  - Lines 706–727: sticky footer (RestTimerBar + spinner or Finish button)

### Notes / follow-ups (not fixed here)
- `message` state (workoutSaved) was dropped — it was set immediately before `router.replace` so was never visible in the UI. This is a minor cleanup, not a behavior change.
- The `useFinishWorkout` hook calls `Alert.alert` internally for PR celebrations. If we ever want to make that testable, we could accept an `onPr` callback — deferred to a future branch.
- `useWorkoutExecutionData` now fetches plan and history in parallel via `Promise.allSettled` (was sequential). This is a performance win with identical behavior.
