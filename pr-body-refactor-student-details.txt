## Task A — Responsibility map (pre-refactor `app/coach/studentDetails.tsx`)

### `useState` / `useRef`
- **None.** No local React state or refs; data came from `useAsyncData` and derived values were plain `const` IIFEs or inline functions after early returns.

### `useEffect` / `useFocusEffect` / `useAsyncData`
- **`useMemo`**: `studentId` from route params (`String(params.studentId ?? "").trim()`).
- **`useCallback`**: `fetcher` for coach-gated load (`studentId`, `userId`, `userRole` deps).
- **`useAsyncData<StudentDetailsData>`**: runs `fetcher` whenever `fetcher` identity changes; exposes `data`, `loading`, `error`.
- **`useMemo`**: `plans` → `detailsData?.plans ?? []`, `logs` → `detailsData?.logs ?? []` (stable empty-array pattern from context-split).

### Async functions and side effects
- **`fetcher`** (inside `useCallback`):  
  - Validates `studentId` and coach role → throws on missing access.  
  - **`await studentService.getStudentById`** → throws if missing or wrong `coachId`.  
  - **`Promise.allSettled`** on: `getLatestTrainingGroupForStudent`, `getWorkoutPlansForStudentAsCoach`, `getWorkoutHistory`; logs partial failures; substitutes `[]` / `null` on rejection (**Phase 1.9**).  
  - Returns `{ student, latestGroup, plans, logs }` (no navigation).

**Note:** The current screen has **no** delete-student, edit-profile, or “View plans / View history” row handlers—those flows are not present on this branch’s file. Navigation is: back, assign workout, create group, view progress, change split, assigned workouts card, dashboard on error.

### Metrics computed inline (extraction targets for Task B)
1. **`lastWorkoutMs` / `lastWorkoutLabel`** — first log’s `completedAt` / `date` via `toMs`; format `toLocaleDateString` month short + day.
2. **`workoutsCompleted`** — `logs.length`.
3. **`avgDurationSeconds` / `avgDurationLabel`** — mean of first ≤10 logs with finite `durationSeconds > 0`, floor; `formatDurationForHistory`.
4. **`compliancePercent`** — rolling 7-day count vs `latestGroup?.workoutsPerWeek`; **null** if target missing/zero/non-finite (**Phase 1.6**); else `round((count7/wpw)*100)` clamped `[0, 999]`.
5. **`currentStreakDays`** — unique local day keys from logs; today + consecutive past days (**Phase 1.7**).
6. **`planById`** — `Map` from `plans` by `id`.
7. **Local `startOfWeekMs`** — Monday 00:00 local for “this week” window (replaced by `startOfWeekMonday` from `dateRanges` in `studentMetrics`, same calendar math).
8. **`weeklyProgress`** — if no `latestGroup.id` or invalid `workoutsPerWeek`, `{ completed: 0, target: wpw, ratio: 0 }`; else count logs in `[weekStart, now]` whose plan’s `groupId === latestGroup.id`; `ratio = clamp(completed/wp w, 0, 1)`.
9. **`assignedProgramPercent`** — if `compliancePercent` is finite, `clamp(round(compliance), 0, 100)`; else `round(ratio * 100)`.
