## Changes

### Task 2.10 — `hooks/useAsyncData<T>`
New hook encapsulating the `let active = true; (async()=>{...})(); return () => { active = false; }` pattern.
- Cancels stale fetches on unmount and dep change via token counter.
- `data` is retained on reload so screens don't flash blank.
- `error` cleared at every fetch start.
- Exposes `reload()` to trigger manual refetches.

Migrated to hook:
- `app/student/workoutExecution.tsx` — fetcher returns `{ plan, priorLogs }`
- `app/coach/studentDetails.tsx` — fetcher returns `{ student, latestGroup, plans, logs }`
- `app/student/workoutHistory.tsx` — fetcher returns `WorkoutLog[]`; pull-to-refresh uses `reload()`

### Task 2.11 — Context split
**Problem:** `useActiveWorkout()` exposed `elapsedSeconds` (ticking 1 Hz) in the same context as stable session state. Every session consumer re-rendered every second.

**Solution:**
- `context/ActiveWorkoutContext.tsx` renamed → `context/ActiveWorkoutSessionContext.tsx`
  - Provider: `ActiveWorkoutSessionProvider`
  - Hook: `useActiveWorkoutSession()` — stable surface (session, mutators, rest-timer state)
  - `elapsedSeconds` removed from this context
- New `context/ElapsedTimeContext.tsx`
  - Provider: `ElapsedTimeProvider` (must live inside `ActiveWorkoutSessionProvider`)
  - Hook: `useElapsedSeconds()` — returns `number`, re-renders at 1 Hz
  - Hook: `useElapsedFormatted()` — thin wrapper for mm:ss string
- Provider tree in `app/_layout.tsx`:
  ```
  <ActiveWorkoutSessionProvider>
    <ElapsedTimeProvider>
      ...
    </ElapsedTimeProvider>
  </ActiveWorkoutSessionProvider>
  ```
- All consumers migrated: `_layout.tsx`, `workoutExecution.tsx`, `workouts.tsx`, `FloatingWorkoutBar.tsx`, `RestTimerBar.tsx`

### Task 1.11 — Debounced AsyncStorage persistence
**Problem:** `persist()` was called on every `updateSet` keystroke (~10 writes/sec).

**Solution:** 500 ms debounce via `persistDebounced()`. Force-flush on critical events:
- `finishSession()` — flushes pending write, then writes `null` immediately
- `AppState` → background/inactive — calls `flushPersist()`
- Provider unmount — calls `flushPersist()` in effect cleanup
- Logout — cancels timer, writes `null` immediately (fire-and-forget)
- Periodic 10s save — cancels debounce, writes `sessionRef.current` directly

**Hard guarantee preserved:** losing the last 500 ms of a keystroke is acceptable; losing the session on force-quit / background is not.
